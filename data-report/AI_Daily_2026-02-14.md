```python
# Multi-Head Attention 核心实现 (chapters/05/attention_measures.py)
class MultiHeadAttention(nn.Module):
    def __init__(self, d_in, d_out, context_length, num_heads, dropout, qkv_bias=False):
        self.num_heads = num_heads
        self.d_out = d_out
        self.head_dim = d_out // num_heads
        self.W_query = nn.Linear(d_in, d_out, bias=qkv_bias)
        self.W_key = nn.Linear(d_in, d_out, bias=qkv_bias)
        self.W_value = nn.Linear(d_in, d_out, bias=qkv_bias)
        self.out_proj = nn.Linear(d_out, d_out)
        self.dropout = nn.Dropout(dropout)
        self.register_buffer("mask", torch.triu(torch.ones(context_length, context_length), diagonal=1))
```

### Issue深度解读

#### PyTorch 2.10 Flex Attention (#957)
**问题背景**：
Flex Attention是PyTorch 2.0引入的动态注意力机制，允许运行时指定注意力掩码模式。在某些边界条件下存在数值误差。

**技术争论**：
```
支持方观点：
- JIT编译优化收益显著 (30%+ 推理加速)
- 动态掩码灵活性高
- 与torch.compile深度集成

反对方：
- 调试复杂度增加
- 边界条件覆盖不完整
- 文档不足
```

**解决方案**：
```python
# 采用掩码补全策略替代直接截断
def flex_attention_with_mask(query, key, value, mask):
    # 保持数值精度
    scores = torch.matmul(query, key.transpose(-2, -1))
    scores = scores.masked_fill(~mask.bool(), float('-inf'))
    return torch.matmul(F.softmax(scores, dim=-1), value)
```
